The Flex Theory

Le module flexbox blablabla expliquer les deux axes à utiliser tout ça tout ça

display

La propriété "display" permet de définir un container comme un flex container, c'est à dire qu'il sera possible d'utiliser flex à l'intérieur de ce container pour disposer des différents éléments comme bon nous semble.

Par défaut, la valeur "flex" dispose les éléments en ligne.

.container {

display: | ;

}

justify-content

La propriété "justify-content" permet d'organiser les éléments sur l'axe principal (défini par la propriété "flex-direction").

On peut ainsi les placer au début, à la fin ou au milieu de cet axe. De plus, cette propriété nous aide à disposer de l'espace restant. Il est en effet possible de répartir l'espace libre entre ces éléments (qui se placeront donc au bord du container) ou autour de ces éléments (générant ainsi un espace égal entre chaque élément et le bord du container)

.container {

display: flex;

justify-content: | | | ;

}

align-items

Tout comme "justify-content", la propriété "align-items" permet de disposer les éléments comme on le désire, mais sur l'axe secondaire cette fois-ci.

Vous remarquerez que les valeurs ne sont pas toutes les mêmes, on dispose cette fois-ci des valeurs baseline et stretch (qui étire les éléments afin qu'ils remplissent tout l'espace disponible).

.container {

display: flex;

align-items: | | | | ;

}

flex-direction

flex-direction est, en général, la première propriété à définir afin de choisir l'axe principal que les éléments devront suivre par défaut dans le container visé.

.container {

display: flex;

flex-direction: | | | ;

}

order

Habituellement, les items sont affichés dans l'ordre dans lequel ils sont intégrés au fichier html, il est donc possible de prévoir leur position selon les propriétés appliquées au flexcontainer.

Avec "order", il vous sera maintenant possible de changer l'ordre dans lequel les items apparaissent dans un flex container sans toucher au fichier html.

.main-item {

order : | | | ;

}

align-self

Cette propriété, placée sur un item, permet d'outrepasser la règle définie par align-items dans son parent. Les mêmes valeurs que pour justify-content sont donc acceptées mais ne s'appliqueront qu'à l'item concerné sans pour autant casser le flow défini dans le container.

.item {

display: flex;

align-self : | | | | ;

}

flex-wrap

Par défaut, les éléments au sein d'un flexcontainer essaieront de tous rentrer dans une seule ligne.

Grâce à la propriété "flex-wrap", vous pouvez permettre aux éléments de se répartir sur plusieurs lignes si l'espace d'une seule ligne n'est pas suffisant pour les contenir proprement.

.container {

display: flex;

flex-wrap : | | ;

}

flex-flow

La propriété flex-flow n'est qu'un raccourcis pour définir en même temps les propriétés flex-direction et flex-wrap.

Soyez attentifs, il est conseillé de n'utiliser les "propriétés doubles" une fois que vous êtes déjà à l'aise avec l'utilisation des propriétés simples.

.container {

display: flex;

flex-flow : direction + wrap value

}

align-content

align-content peut être vu comme l'équivalent de "justify-content" mais cette fois-ci sur l'axe secondaire du container. Ces deux propriétés acceptent donc les mêmes valeurs.

.container {

display: flex;

align-content : | | | | | ;

}
