The Grid Theory

Grid est un ensemble de propriétés CSS qui permet la disposition d'éléments dans une grille sur une page web. Les premières spécifications de CSS Grid ont été proposées par Microsoft en 2011. Comme son nom l'indique, on parle d'une grille en deux dimensions. Les lignes et les colonnes ainsi que leur comportement sont définis entièrement en CSS.

1 Définir une grille

display

La propriété "display" permet de définir un container comme un grid container, c'est à dire qu'il sera possible d'utiliser grid à l'intérieur de ce container pour disposer des différents éléments comme bon nous semble.

Pour voir le placement par défaut de "grid", il suffit de créer une grille avec un ensemble d'objets. Sans fournir aucune information de placement, ces objets se placeront chacun sur une cellule de la grille.

.container

display: | ;

grid-template-columns: 1fr 1fr 1fr 1fr;

grid-template-rows: 1fr 1fr 1fr;

}

grid-template-columns

"grid-template-columns" permet de créer des colonnes.

On peut choisir la largeur de chacunes de nos colonnes (definie par des px (pixels), % (poucentages), fr (fractions) ou auto (automatic) ). Ainsi 1fr 1fr 1fr ou repeat(3, 1fr) permettent d'obtenir une grille de 3 colonnes faisant chacune 1/3 de l'espace disponible restant (après calcul des marges et paddings).

.container {

display: grid;

grid-template-columns: | | | | ;

grid-template-rows: 1fr 1fr 1fr;

}

grid-template-areas

La propriété "grid-template-areas" permet de définir des zones de grille nommées comme on le souhaite. Chaque cellule a un nom qui peut être attribué à un élément en css.

.container {

display: grid;

grid-template-areas: |

;}

div.A { grid-area : un; }

div.B { grid-area : deux; }

div.C { grid-area : trois; }

div.D { grid-area : quatre; }

gaps

"gaps" va nous servir à définir des espaces entre les cellules que comporte la grille, ce qui peut s'avérer utile si l'on veux aérer notre page.

.container {

display: grid;

grid-template-columns: 1fr 1fr 1fr 1fr;

grid-template-rows: 1fr 1fr 1fr;

row-gap: value ;

column-gap: value ;

gap: value ;

}

2 Placer nos Div dans la grille

Attention les propriétés qui suivent sont faites pour affecter les divisions que vous désirez placer dans une grille que vous avez définie auparavant.

Les éléments auxquels on ne donne pas d'ordres se placeront d'eux-même dans la grille.

grid-row-start / grid-row-end / grid-row

"grid-row-start" définit la ligne sur laquelle la division va commencer. Son antagoniste, "grid-row-end" permet de choisir là où elle devra s'arrêter.

"grid-row" permet de définir ces deux valeurs dans une seule propriété en les séparant d'un "/".

Il est bon de savoir que plutôt que de définir un "grid-row-end" en seconde valeur de "grid-row", il est possible de simplement lui indiquer sur combien de ligne la division doit se positionner à l'aide de "span".

"grid-row : 1 / span 3" mettra la division concernée sur la première ligne et la fera courrir sur trois lignes.

.item {

grid-row-start : | | | ;

grid-row-end : | | | ;

grid-row : | | |

}

grid-column-start / grid-column-end / grid-column

"grid-column-start" et "grid-column-end" fonctionnent exactement de la même manière que pour les lignes.

Ainsi, comme pour les lignes, il est possible de fusionner les deux propriétés pour n'utiliser que "grid-column" et définir en une seule ligne le début et la fin de l'emplacement que doit prendre la division dans la grille.

.item {

grid-column-start : | | | ;

grid-column-end : | | | ;

grid-column : | | |

}

grid-area

"grid-area" permet de combiner en une seule ligne "grid-row" et "grid-column".
"grid-area" prend donc 4 valeurs en entrée qui doivent toutes être séparées par des "/"

.item {

grid-area : | | | ;

}
